# 电子证书跨机构可信验证平台流程图说明版

## 1. 编写目的

本文档用于配合系统流程图展示平台主要业务流程。相较于详细功能文档，本文档更强调流程节点清晰、前后顺序明确，适合直接用于毕业设计论文插图说明、答辩 PPT 讲解和流程图文字标注。

## 2. 系统总流程说明

系统总流程可概括为：

开始 → 用户注册或登录 → 进入工作台 → 选择业务功能 → 执行证书发布或验证 → 返回结果 → 记录日志 → 结束

对应说明如下：

1. 用户进入系统后，首先完成注册或登录
2. 登录成功后进入系统工作台
3. 根据当前角色选择证书发布、草稿审批、证书管理、链上验证或授权码管理
4. 若进行证书发布，则根据角色进入草稿提交流程或直接发布流程
5. 若进行证书验证，则根据输入方式进入链上验证流程或文件验真流程
6. 验证结束后系统输出结果，并写入验证日志
7. 流程结束

## 3. 注册与登录流程图说明

### 3.1 注册流程

流程图节点建议写法如下：

开始 → 输入邮箱 → 发送验证码 → 填写注册信息 → 校验验证码 → 写入用户信息 → 注册成功 → 结束

流程说明如下：

1. 用户输入邮箱地址并请求发送验证码
2. 系统生成验证码并发送邮件
3. 用户填写用户名、密码、邮箱、验证码、机构和角色
4. 后端校验验证码是否正确、是否过期
5. 校验通过后对密码加密并写入用户表
6. 返回注册成功结果

### 3.2 登录流程

流程图节点建议写法如下：

开始 → 输入用户名和密码 → 校验账号信息 → 生成 JWT → 登录成功 → 进入工作台 → 结束

流程说明如下：

1. 用户输入用户名和密码
2. 系统查询数据库中的用户记录
3. 校验密码是否正确
4. 登录成功后生成 JWT 令牌
5. 前端保存登录态并进入业务页面

## 4. 证书发布流程图说明

证书发布分为两条主线：普通用户草稿发布和管理员直接发布。

### 4.1 OCR 辅助录入流程

流程图节点建议写法如下：

开始 → 上传证书图片 → OCR 识别 → 提取文字字段 → 回填表单 → 人工修正 → 结束

流程说明如下：

1. 用户上传证书图片
2. 系统识别图片中的文字
3. 自动提取证书名称、持证人、编号、机构和日期等信息
4. 将识别结果填入表单
5. 用户确认或补充修改后进入正式发布流程

### 4.2 普通用户草稿发布流程

流程图节点建议写法如下：

开始 → 填写证书信息 → 上传原始文件 → 提交发布请求 → 判断角色是否为普通用户 → 写入草稿表 → 状态设为待审批 → 结束

流程说明如下：

1. 普通用户填写证书信息并上传原始文件
2. 系统识别用户角色为普通用户
3. 后端校验所属机构是否完整
4. 将证书信息和文件写入草稿表
5. 草稿状态设为待审批
6. 返回草稿提交成功结果

### 4.3 管理员直接发布流程

流程图节点建议写法如下：

开始 → 填写证书信息 → 上传原始文件 → 计算文件哈希 → 检查是否重复 → 上传 IPFS → 写入证书表 → MetaMask 上链 → 保存交易哈希 → 结束

流程说明如下：

1. 管理员填写证书信息并上传原始文件
2. 后端计算文件 SHA256 哈希
3. 系统检查是否已有相同内容证书
4. 若无重复，则上传文件到 IPFS
5. 将证书信息写入正式证书表
6. 前端连接 MetaMask 并调用合约完成上链登记
7. 交易确认后保存交易哈希
8. 发布成功

## 5. 草稿审批流程图说明

流程图节点建议写法如下：

开始 → 管理员进入草稿箱 → 查询待审批草稿 → 选择草稿 → 确认发布 → 生成正式证书 → 上链登记 → 更新草稿状态 → 结束

流程说明如下：

1. 管理员进入草稿箱页面
2. 系统查询当前机构下的待审批草稿
3. 管理员选择某条草稿执行确认发布
4. 后端将草稿状态改为处理中
5. 系统读取草稿信息并生成正式证书
6. 证书文件上传 IPFS，证书信息写入正式证书表
7. 前端继续调用 MetaMask 完成上链
8. 草稿状态改为已审批，流程结束

## 6. 证书查询与管理流程图说明

流程图节点建议写法如下：

开始 → 输入查询条件 → 提交查询 → 按权限过滤数据 → 返回证书列表 → 查看详情或管理 → 结束

流程说明如下：

1. 用户在证书列表页输入查询条件
2. 前端发起查询请求
3. 后端根据角色和机构范围过滤数据
4. 数据库返回符合条件的证书列表
5. 前端展示分页结果
6. 管理员可执行查看、修改或删除操作

## 7. 证书链上验证流程图说明

流程图节点建议写法如下：

开始 → 输入证书编号或哈希 → 定位证书 → 调用合约验证 → 获取链上结果 → 返回验证信息 → 记录日志 → 结束

流程说明如下：

1. 用户输入证书编号或证书哈希
2. 后端定位目标证书并获取真实哈希
3. 系统调用智能合约执行只读验证
4. 合约返回 IPFS 地址、发布者地址和上链时间
5. 后端整理验证结果并返回给前端
6. 前端展示结果并记录验证日志

## 8. 文件真实性验证流程图说明

流程图节点建议写法如下：

开始 → 上传证书文件 → 计算文件哈希 → 查询数据库 → 查询链上记录 → 判断验证结果 → 返回结果 → 记录日志 → 结束

流程说明如下：

1. 用户上传待验证证书文件
2. 后端重新计算文件 SHA256 哈希
3. 使用该哈希查询数据库中的证书记录
4. 若数据库无记录，则说明文件不可信或已被篡改
5. 若数据库有记录但链上无记录，则结果为未上链
6. 若数据库记录和链上记录一致，则结果为真实
7. 前端展示最终结果，并记录验证日志

## 9. 跨机构授权验证流程图说明

### 9.1 授权码生成流程

流程图节点建议写法如下：

开始 → 总管理员登录 → 设置有效期 → 生成授权码 → 旧码失效 → 保存授权码哈希 → 返回明文授权码 → 结束

流程说明如下：

1. 总管理员进入授权码管理页面
2. 设置授权码有效时长
3. 后端生成新的授权码
4. 系统使旧授权码失效，仅保留最新有效码
5. 将授权码哈希和过期时间保存到数据库
6. 返回授权码明文供管理员分发

### 9.2 授权验证流程

流程图节点建议写法如下：

开始 → 发起验证请求 → 是否携带授权码 → 校验授权码 → 返回明文结果或加密结果 → 记录日志 → 结束

流程说明如下：

1. 用户发起链上验证或文件验真请求
2. 请求中可选择是否携带授权码
3. 后端校验授权码是否有效
4. 若有效，则返回明文验证结果
5. 若无效或未提供，则返回加密后的验证内容
6. 系统写入日志后结束流程
