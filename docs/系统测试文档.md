# 电子证书跨机构可信验证平台系统测试文档

## 1. 文档概述

本文档用于说明电子证书跨机构可信验证平台的测试目标、测试环境、测试内容、测试方法和测试结果。系统测试的主要目的是验证平台各模块是否能够按照设计要求正常运行，并检查用户注册登录、证书发布、草稿审批、链上登记、证书验证、授权控制和日志留痕等核心功能是否满足预期。

本系统属于集前端、后端、数据库、IPFS 和区块链于一体的综合型应用，因此测试内容不仅包括普通 Web 功能测试，还包括链上链下协同验证、文件真实性比对和跨机构授权控制等内容。

## 2. 测试目标

本系统测试的主要目标如下：

- 验证各功能模块是否能够正常运行
- 验证系统主要业务流程是否完整闭环
- 验证不同角色的权限控制是否正确
- 验证区块链登记与链上查询功能是否可用
- 验证文件真实性检测结果是否准确
- 验证授权码机制是否能够正确控制验证结果返回方式
- 验证系统在异常输入和错误场景下是否能够给出合理提示

## 3. 测试环境

### 3.1 硬件环境

- 开发设备：个人计算机
- 处理器：常规 x64 架构处理器
- 内存：8GB 及以上
- 网络环境：可访问 MySQL、本地服务、IPFS 网关与区块链测试网络

### 3.2 软件环境

- 操作系统：Windows
- 前端运行环境：Node.js + Vite
- 后端运行环境：Node.js + Express
- 数据库：MySQL
- 区块链开发环境：Hardhat
- 钱包工具：MetaMask
- 测试网络：Sepolia
- 文件存储服务：Pinata / IPFS
- OCR 服务：百度 OCR

### 3.3 测试方法

本项目主要采用以下测试方法：

- 功能测试
- 接口测试
- 角色权限测试
- 流程测试
- 异常测试
- 结果比对测试

## 6. 关键测试过程说明

### 6.1 用户注册登录测试结果分析

测试表明，系统能够正确完成邮箱验证码发送、注册信息校验、密码加密存储和 JWT 登录令牌签发。在错误验证码、重复注册和错误密码等异常场景下，系统也能够返回明确提示，说明用户认证模块实现较为稳定。

### 6.2 草稿审批流程测试结果分析

测试表明，普通用户提交证书后，数据能够正确写入草稿表并进入待审批状态；管理员能够查看所属机构草稿并完成审批；草稿审批后能够成功转为正式证书记录。该结果说明系统已实现符合业务逻辑的审批发布机制。

### 6.3 链上登记与验证测试结果分析

测试结果表明，管理员发布证书后，前端能够通过 MetaMask 正常发起交易，智能合约能够完成证书哈希登记，并返回有效交易哈希。后续通过证书哈希或证书编号进行链上验证时，系统能够正确返回 IPFS 地址、发布者地址和时间戳，说明链上链下协同逻辑有效。

### 6.4 文件真实性验证测试结果分析

测试表明，系统能够通过重新计算文件哈希的方式准确判断文件状态，并输出真实、未上链和被篡改三类结果。该结果说明文件真实性验证模块具有较强实用性，能够为用户直接提供文件级验真能力。

### 6.5 授权控制测试结果分析

测试结果表明，总管理员生成授权码后，验证请求在携带有效授权码时能够获取明文结果，而未授权请求只能获取加密内容。该结果说明系统在跨机构验证场景下具备一定的访问控制能力。

## 7. 异常与安全性测试说明

为进一步验证系统稳定性，还对以下异常场景进行了测试：

- 重复发送邮箱验证码
- 验证码错误或过期
- 用户未登录访问受限接口
- 普通用户尝试执行管理员操作
- 非总管理员尝试生成授权码
- 重复证书发布
- 重复链上登记
- 无效文件上传
- 不存在证书的验证请求

测试结果表明，系统在上述异常场景下均能够进行有效拦截，并返回较清晰的错误提示，说明系统具备基本的安全防护与异常处理能力。

## 8. 测试结果总结

综合测试结果可以看出，本系统主要功能模块均能够按照预期运行，具体表现为：

- 用户注册、登录和认证功能正常
- 普通用户草稿提交与管理员审批流程完整
- 管理员直接发布证书流程正常
- IPFS 上传、链上登记和交易回写功能可用
- 证书链上验证和文件真实性验证结果准确
- 跨机构授权机制能够控制验证结果返回方式
- 验证日志能够正常记录

从测试结果来看，系统已基本达到电子证书可信发布与跨机构验证平台的设计目标，能够满足毕业设计原型系统的功能要求。

## 9. 本章小结

本文档对电子证书跨机构可信验证平台进行了较系统的测试说明。测试结果表明，系统在用户管理、证书发布、草稿审批、链上登记、文件验真、授权验证和日志留痕等方面均具有较好的可用性和正确性。虽然系统在大规模并发、生产级安全防护和复杂组织治理方面仍有进一步优化空间，但从毕业设计原型实现角度看，系统已经具备较完整的业务闭环和较好的测试表现。
