# 智能合约说明

## 1. 概述

`CertificateRegistry` 合约部署在 Ethereum Sepolia 测试网上，用于为电子证书提供链上可信索引。合约存储 `certHash → (ipfsHash, issuer, timestamp, revoked)` 的映射关系，任何人均可只读验证，但写入操作受访问控制约束。

合约源码：`contracts/contracts/CertificateRegistry.sol`

## 2. 数据结构

```solidity
struct Certificate {
  string certHash;     // 证书文件 SHA256 哈希
  string ipfsHash;     // IPFS 存储 CID
  address issuer;      // 发行人钱包地址
  uint256 timestamp;   // 上链时间戳
  bool revoked;        // 是否已撤销
}
```

## 3. 状态变量

| 变量 | 类型 | 可见性 | 说明 |
|------|------|--------|------|
| `owner` | `address` | `public` | 合约所有者，部署者自动成为 owner |
| `authorizedIssuers` | `mapping(address => bool)` | `public` | 授权发行人白名单 |
| `certificateCount` | `uint256` | `public` | 链上证书总数 |
| `certificates` | `mapping(string => Certificate)` | `private` | 证书存储映射 |

## 4. 事件

### CertificateStored

```solidity
event CertificateStored(
  string indexed certHashIndexed,
  string certHash,
  string ipfsHash,
  address indexed issuer,
  uint256 timestamp
);
```

- 每次成功存储证书时触发
- `certHashIndexed` 和 `issuer` 为 indexed 参数，可高效过滤日志

### CertificateRevoked

```solidity
event CertificateRevoked(
  string indexed certHashIndexed,
  string certHash,
  address indexed revokedBy,
  uint256 timestamp
);
```

- 证书被撤销时触发

### IssuerAuthorized / IssuerRevoked

```solidity
event IssuerAuthorized(address indexed issuer);
event IssuerRevoked(address indexed issuer);
```

### OwnershipTransferred

```solidity
event OwnershipTransferred(address indexed previousOwner, address indexed newOwner);
```

## 5. 访问控制

| 修饰符 | 条件 |
|--------|------|
| `onlyOwner` | `msg.sender == owner` |
| `onlyAuthorized` | `msg.sender == owner` 或 `authorizedIssuers[msg.sender] == true` |

- 部署者自动成为 `owner` 并加入白名单
- `transferOwnership` 会将新 owner 自动加入白名单
- 不允许撤销 owner 自身的 issuer 权限

## 6. 函数接口

### 6.1 storeCertificate

```solidity
function storeCertificate(string calldata certHash, string calldata ipfsHash) external onlyAuthorized
```

- **用途**：存储单个证书的链上索引
- **权限**：仅授权发行人
- **约束**：certHash 和 ipfsHash 不能为空；certHash 不能重复
- **事件**：触发 `CertificateStored`

### 6.2 storeCertificateBatch

```solidity
function storeCertificateBatch(
  string[] calldata certHashes,
  string[] calldata ipfsHashes
) external onlyAuthorized
```

- **用途**：批量存储证书（单次最多 50 条）
- **权限**：仅授权发行人
- **行为**：已存在的 certHash 自动跳过，不会 revert
- **事件**：每条成功存储的证书触发 `CertificateStored`

### 6.3 revokeCertificate

```solidity
function revokeCertificate(string calldata certHash) external
```

- **用途**：撤销证书
- **权限**：仅证书原始发行人或合约 owner
- **约束**：证书必须存在且未被撤销
- **事件**：触发 `CertificateRevoked`

### 6.4 verifyCertificate

```solidity
function verifyCertificate(string calldata certHash)
  external view
  returns (string memory ipfsHash, address issuer, uint256 timestamp)
```

- **用途**：查询证书基本信息（向后兼容）
- **权限**：任何人
- **说明**：不返回 revoked 状态

### 6.5 verifyCertificateEx

```solidity
function verifyCertificateEx(string calldata certHash)
  external view
  returns (string memory ipfsHash, address issuer, uint256 timestamp, bool revoked)
```

- **用途**：查询证书完整信息（含撤销状态）
- **权限**：任何人

### 6.6 certificateExists

```solidity
function certificateExists(string calldata certHash) external view returns (bool)
```

- **用途**：快速检查证书是否存在
- **权限**：任何人
- **说明**：仅返回 bool，gas 消耗低于 verifyCertificate

### 6.7 authorizeIssuer

```solidity
function authorizeIssuer(address issuer) external onlyOwner
```

- **用途**：将地址加入授权发行人白名单
- **事件**：触发 `IssuerAuthorized`

### 6.8 revokeIssuer

```solidity
function revokeIssuer(address issuer) external onlyOwner
```

- **用途**：将地址从白名单移除
- **约束**：不能撤销 owner 自身
- **事件**：触发 `IssuerRevoked`

### 6.9 transferOwnership

```solidity
function transferOwnership(address newOwner) external onlyOwner
```

- **用途**：转移合约所有权
- **行为**：新 owner 自动获得 issuer 权限
- **事件**：触发 `OwnershipTransferred`

## 7. 测试覆盖

合约配套 33 个 Hardhat 单元测试，覆盖以下场景：

| 测试组 | 用例数 | 覆盖范围 |
|--------|--------|----------|
| Deployment | 3 | owner 设置、issuer 初始化、计数初始值 |
| Access Control | 7 | 授权/撤销/转移所有权、边界条件 |
| storeCertificate | 6 | 正常存储、授权存储、未授权拒绝、重复拒绝、空值拒绝 |
| storeCertificateBatch | 6 | 批量存储、跳过重复、长度不匹配、空批次、超限、未授权 |
| verifyCertificate | 2 | 正常查询、不存在 revert |
| verifyCertificateEx | 2 | 活跃状态、撤销后状态 |
| revokeCertificate | 5 | issuer 撤销、owner 撤销、陌生人拒绝、重复撤销、不存在 |
| certificateExists | 2 | 不存在返回 false、存在返回 true |

运行测试：

```bash
cd contracts
npm test
```

## 8. 部署

```bash
cd contracts
npm run deploy:sepolia
```

部署脚本会输出：
- 合约地址
- 合约 owner 地址
- 需要更新的环境变量

部署后需更新：
- `backend/.env` 中的 `CERT_REGISTRY_ADDRESS`
- `frontend/.env` 中的 `VITE_CERT_REGISTRY_ADDRESS`

## 9. 与系统集成

### 前端调用路径

1. 用户通过 MetaMask 连接钱包并切换到 Sepolia 网络
2. 前端使用 `ethers.BrowserProvider` + `signer` 构造合约实例
3. 调用 `storeCertificate` 发起上链交易
4. 交易确认后将 `tx_hash` 通过后端 API 回写数据库

### 后端调用路径

1. 后端通过 `ethers.JsonRpcProvider` + RPC URL 构造只读 provider
2. 调用 `verifyCertificate` 进行链上验证
3. 将链上结果与数据库记录对比，返回验证结论
